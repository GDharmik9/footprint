Here is the complete unified technology stack utilized across the **Footprint** monorepo workspace and cloud deployment architecture:

---

### 💻 Local Development & Workspace Architecture

* **Monorepo Package Manager:** NPM Workspaces (v10+)
* **Primary Language:** TypeScript (v5.3+) — Strict type compilation enforced end-to-end
* **Build Tooling & Bundler:** Vite (v5.1+) — Fast Hot Module Replacement (HMR) for frontend tooling
* **Runtime Monitoring:** `tsc-watch` — Automatic backend compilation and server restarts on source file modifications
* **Process Concurrency:** `concurrently` (v8.2) — Simultaneous orchestration of API and Frontend servers with a single CLI command

---

### 🎨 Frontend Client Application (`apps/web`)

* **Core UI Library:** React (v19.0) — Declarative state management and component hooks
* **Styling Engine:** Tailwind CSS — Utility-first, performance-optimized visual layouts
* **Design Tokens & Typography:** Custom CSS HSL Palettes combined with Google Fonts integration (**Outfit** for high-impact displays, **Inter** for UI data legibility)
* **Dynamic Visuals:** Native SVG Engineering — Scalable vector components responding programmatically to user experience points (Leaves) levels
* **Data Strategy:** Fetch API — Promise-based HTTP client featuring automated fallback mechanisms for localized client-side simulation when the API server is unavailable

---

### ⚙️ Backend Core & Services (`apps/api` & `packages/`)

* **Server Framework:** Express.js (v4.19) — REST API routing, server configurations, and endpoint processing
* **Calculation Engine:** `@footprint/carbon-math` — Custom functional arithmetic pipeline built according to international Greenhouse Gas (GHG) Protocol methodologies
* **Shared Type System:** `@footprint/shared-types` — Unified interface boundaries exported to prevent structural drifting between the client UI and backing models
* **Cross-Origin Policy:** `cors` middleware — Configured to securely handle cross-origin scripting access boundaries between local execution instances

---

### ☁️ Enterprise Production Google Cloud (GCP) Infrastructure

* **Security & Gatekeeping:** Google Cloud API Gateway — Token authentication filtering, routing, and endpoint rate-limiting
* **Serverless Compute:** Cloud Run — Scalable serverless Docker container layer hosting the backing microservices (auto-scales to zero based on current execution demand)
* **Asynchronous Message Bus:** Cloud Pub/Sub — Distributed enterprise message queue handling data streams generated from transit SDKs and utility connectors
* **Queue Orchestration:** Cloud Tasks & Cloud Scheduler — Rate-limited target task queues mapped alongside cron-triggers for daily calculation tasks and league updates
* **Operational Database:** Cloud SQL (PostgreSQL) — Managed persistent relational relational store mapping structural data for accounts, streaks, and B-Corp partners
* **Metric Ingestion & Analytics:** TimescaleDB on Compute Engine (or Bigtable) — High-throughput database architecture tracking real-time carbon telemetry rows
* **High-Speed Caching:** Memorystore for Redis — Low-latency memory storage caching grid carbon coefficients by geographic region (preventing downstream API timeouts)
* **Credential Defense:** Secret Manager — Hardened storage vault handling database credentials, service account properties, and private API access tokens

---

### 🔌 Third-Party API Integrations

* **Utility Infrastructure Automation:** Arcadia API (or Urjanet) — Standardized account validation abstraction supplying monthly $kWh$ and gas parameters
* **Smart Home Environment:** Google Nest Device Access API — Secure telemetry verification fetching target smart thermostat configurations and Eco-Mode statuses
* **Edge Telemetry Tracking:** Radar.io SDK & API — Edge processing distance calculation framework running locally on mobile devices without server-side storage of route coordinates
* **Environmental Data Math:** Electricity Maps API & EPA eGRID — Dynamic databases tracking real-time grid mix emissions and structural baseline indices grouped by regional zip boundaries
* **Ecosystem Verification:** Eden Reforest Projects Webhooks — Automated distribution pipelines handling financial fulfillment on user reward redemptions
